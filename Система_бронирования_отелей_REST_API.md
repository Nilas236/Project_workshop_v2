# Система бронирования отелей (REST API)

Асинхронный REST API для системы бронирования отелей, разработанный в рамках проектного практикума. Проект демонстрирует применение современных стандартов веб-разработки, многослойной архитектуры и принципов разделения ответственности.

## 🚀 Основной функционал

- **Управление отелями и номерами:** CRUD-операции, сложный поиск доступных отелей по заданным датам.
- **Бронирование:** Создание, просмотр и отмена бронирований для авторизованных пользователей.
- **Авторизация и безопасность:** Регистрация и аутентификация на основе JWT-токенов, хранящихся в HTTP-only cookies.
- **Фоновые задачи:** Асинхронная обработка изображений и рассылка email-уведомлений с использованием Celery и Redis.
- **Кэширование:** Использование Redis для ускорения ответов API.

## 🛠 Технологический стек

| Технология | Назначение |
|---|---|
| **FastAPI** | Асинхронный веб-фреймворк для создания API |
| **PostgreSQL** | Основная реляционная база данных |
| **SQLAlchemy (async)** | ORM для взаимодействия с базой данных |
| **Alembic** | Управление миграциями схемы БД |
| **Pydantic v2** | Строгая валидация данных и управление конфигурацией |
| **Redis** | In-memory хранилище для кэширования и брокер задач |
| **Celery** | Очередь для выполнения асинхронных и периодических задач |
| **pytest** | Автоматизированное тестирование с изолированной БД |

## 🏗 Архитектура проекта

Проект построен по принципу разделения ответственности (слоистая архитектура), что обеспечивает масштабируемость и легкость тестирования.

- `src/api/` — Роутеры FastAPI (эндпоинты).
- `src/models/` — ORM-модели SQLAlchemy.
- `src/schemas/` — Pydantic-схемы (валидация входа/выхода).
- `src/repositories/` — Паттерн Repository (абстракция над БД).
- `src/services/` — Бизнес-логика приложения.
- `src/tasks/` — Фоновые задачи Celery.
- `src/connectors/` — Интеграции с внешними сервисами.
- `src/utils/` — Вспомогательные утилиты (в т.ч. реализация Unit of Work).
- `tests/` — Автотесты на базе pytest.

## ⚙️ Установка и запуск (Локально)

### 1. Клонирование репозитория
```bash
git clone https://github.com/Nilas236/Project_workshop_v2.git
cd Project_workshop_v2
```

### 2. Настройка окружения
Создайте виртуальное окружение и установите зависимости:
```bash
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
# venv\Scripts\activate   # Для Windows
pip install -r requirements.txt
```

Создайте файл `.env` в корне проекта и задайте следующие переменные:
```env
MODE=DEV
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASS=your_password
DB_NAME=hotel_booking
REDIS_HOST=localhost
REDIS_PORT=6379
JWT_SECRET_KEY=your_super_secret_key
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

### 3. Запуск инфраструктуры
Убедитесь, что у вас локально или в Docker запущены **PostgreSQL** и **Redis**.

### 4. Применение миграций
Создайте таблицы в базе данных:
```bash
alembic upgrade head
```

### 5. Запуск приложения
Запустите сервер FastAPI:
```bash
uvicorn src.main:app --reload
```
- API будет доступно по адресу: `http://127.0.0.1:8000`
- Интерактивная документация (Swagger UI): `http://127.0.0.1:8000/docs`

### 6. Запуск Celery (в отдельных терминалах)
Запуск воркера для обработки задач:
```bash
celery -A src.tasks.celery_app:celery_instance worker --loglevel=INFO
```
Запуск планировщика (Beat) для периодических задач:
```bash
celery -A src.tasks.celery_app:celery_instance beat --loglevel=INFO
```

## 🧪 Тестирование

Для запуска тестов используется отдельная тестовая база данных. Убедитесь, что настроен файл `.env-test`.
Запуск тестов:
```bash
pytest -v
```
